Data Ingestion — Trade Detection
Three parallel detection streams feeding into unified trade queue
3 modules ▼
🔌
CLOB WebSocket
src/monitor.ts
Real-time price stream from Polymarket CLOB. Detects price movements and triggers immediate Data API polling for near-instant trade detection.
📡 connect() — wss://ws-subscriptions-clob.polymarket.com/ws/market
📨 handleWsMessage() — Parse last_trade_price events
⚡ triggerImmediatePoll() — 500ms debounce, 100ms batch delay
🔄 subscribe() — Subscribe to all target market asset IDs
→ Data API (immediate poll)
→ pollAllWallets()
🔄
REST Polling
src/monitor.ts
Polls Polymarket Data API every 3 seconds for each target wallet. Primary trade detection mechanism. Deduplicates via seenTradeIds Set.
📊 pollWallet() — GET /activity?user={address}&limit=20
🔍 parseTrade() — Convert DataApiTrade → ParsedTrade
🛡️ seenTradeIds — Deduplication Set (prevents double-processing)
🔒 polling guard — Prevents concurrent pollAllWallets()
← WebSocket (trigger)
→ onTrade callback
⛓️
On-Chain Settlement Monitor
src/onchain.ts
Monitors CTF Exchange & NegRiskCtfExchange contracts on Polygon. Verifies trade settlement. Slower than CLOB — used for verification only.
📝 OrderFilled events — CTFExchange + NegRiskCtfExchange
🔎 matchTarget() — Check maker/taker against target wallets
🔄 scheduleReconnect() — Auto-reconnect on WS drop (10s delay)
→ onFill callback (verification)
← WS_RPC_URL (Alchemy/Infura)

2
Trade Processing Pipeline
Filter → AI Analysis → Risk Check → Execute — the core decision chain
4 modules ▼
🎯
Trade Handler / Router
src/index.ts
Central orchestrator. Receives detected trades, runs the full pipeline: skip BUY-only trades → AI filter → risk check → execute copy trade.
🚫 Skip BUY trades — Only copy SELL signals from whales
🤖 AI Filter Gate — if (aiFilter) → analyzeTrade()
🛡️ Risk Gate — checkTrade() before execution
🚀 Execute Copy — Execute copy trade via CLOB API
🧠
AI Trade Filter
src/ai-filter.ts
LLM-powered analysis. Estimates true probability, compares to market price. Supports OpenAI, Anthropic, OpenRouter, or custom endpoints. Fetches market context + cross-platform consensus in parallel.
📊 fetchMarketContext() — Polymarket API: volume, liquidity, history
🌐 fetchConsensus() — Manifold + Kalshi + FinFeedAPI (parallel)
💬 callLLM() — Prompt with context → probability estimate
✅ Ensemble voting — Dual-model consensus (optional secondModel)
💾 Cache (15min TTL) — Avoids re-analyzing same market
⚡ Rate limiter — Max 10 calls/min, 30s timeout
← Trade data
→ AIFilterResult (approve/reject)
← cross-platform.ts
← proxy.ts (fetch)
🛡️
Risk Management
src/risk.ts
Multi-layer risk controls. Blocks trades that violate limits. Tracks session P&L, drawdown, consecutive losses, per-market caps.
📏 Position sizing — multiplier × target size, clamped to min/max
🚫 Session notional cap — Max $1,000 total (configurable)
🚫 Per-market cap — Max $200 per market (configurable)
📉 Daily loss limit — 5% daily loss → halt (configurable)
🛑 Max drawdown — 25% from peak → halt (configurable)
💀 Total loss halt — 40% total loss → permanent halt
🚀
Trade Execution
src/executor.ts
Executes copy trades via Polymarket CLOB API. Signs orders with EIP-712, handles FOK/GTC/FAK order types, tracks fills.
📝 createOrder() — Build signed CLOB order
🔐 signOrder() — EIP-712 signature with private key
✅ submitOrder() — POST to CLOB /order endpoint
📊 getOrderBook() — Check liquidity before sizing
Trade Executed → Notify & Track

🔍 Trade Detection
❓ New trade detected from target wallet?
✓ Trade in seenTradeIds?
→ Skip (already processed)
✗ New trade
→ Route to processing pipeline
🚫 Trade Side Filter
❓ Trade side is SELL?
✓ SELL trade — whale is exiting
→ Continue to AI filter
✗ BUY trade
→ Skip (log only, don't copy buys)
🧠 AI Filter
❓ AI_FILTER_ENABLED?
✓ Enabled — run LLM analysis
❓ Cache hit (within 15min)?
→ Use cached result
❓ probability × confidence > minEdge?
→ ✅ Approve trade
→ ❌ Reject trade
✗ Disabled
→ Skip AI, go to risk check
⚡ AI Error Handling
❓ LLM call failed?
✓ Error occurred
❓ FAIL_OPEN = true?
→ ✅ Approve anyway (fail-open)
→ ❌ Reject trade (fail-closed)
✗ No error
→ Continue normal flow
🛡️ Risk Check
❓ Halted? (drawdown/daily loss/total loss)
✓ Bot is halted
→ 🛑 Block all trades
✗ Not halted
❓ Session notional + trade > cap?
→ 🚫 Block (session cap)
❓ Per-market notional > cap?
→ 🚫 Block (market cap)
→ ✅ Allow — calculate position size
🚀 Execution
❓ DRY_RUN = true?
✓ Simulation mode
→ Log simulated fill, update virtual positions
✗ Live mode
❓ Orderbook has liquidity?
→ Skip (insufficient liquidity)
→ Sign order (EIP-712) → POST /order → track fill

Risk Severity Mitigation Module Action
Excessive losses Critical Daily loss limit (5%), max drawdown (25%), total loss halt (40%) risk.ts Halt all trading
Oversized positions High Position multiplier, min/max trade size, per-market cap ($200) risk.ts Block trade
Session overexposure High Session notional cap ($1,000 default) risk.ts Block trade
AI filter failure Medium Fail-open (approve on error), fail-closed option, 30s timeout ai-filter.ts Approve or reject based on config
Bad AI probability Medium Min confidence threshold (60%), min edge (5%), ensemble voting ai-filter.ts Reject low-confidence trades
API rate limiting Low AI: 10 calls/min. Telegram: 20 msg/min. Proxy support. ai-filter.ts, telegram.ts Queue, debounce, cache
WebSocket disconnect Medium Auto-reconnect (10s), REST polling as backup, polling guard monitor.ts, onchain.ts Reconnect + fallback
Duplicate trades Low seenTradeIds Set for deduplication monitor.ts Skip duplicate
Concurrent polling Low polling guard flag with try/finally monitor.ts Skip concurrent poll
Telegram parse errors Low escapeMarkdown() for legacy mode, plain text fallback telegram.ts Retry as plain text
Network timeout Medium AbortSignal.timeout() on all fetch calls (8-30s) all modules Abort + log + continue
Stale market data Low AI cache TTL (15min), real-time orderbook check before execution ai-filter.ts, executor.ts Refresh or skip
Insufficient liquidity Medium Orderbook check before sizing, slippage tolerance (2%) executor.ts Skip trade
Cross-platform data skew Low FinFeedAPI only queries Polymarket (Kalshi native). Spread detection. cross-platform.ts Log high-spread warning
